# Component Catalog

This is a comprehensive catalog of all components available in the Voltrux Motors system. Each component is designed to follow best practices and maintain consistency across the platform.

## Available Components

### Core Components

- **[Article List](./article-list)** - Displays articles with featured content, category filtering, and responsive design
- **[Header](./header)** - Main navigation with logo, navigation links, call-to-action button, and mobile-friendly navigation
- **[Test Drive Form](./test-drive)** - Comprehensive form for scheduling test drives with location, model, and time selection
- **[Truck Lister](./truck-lister)** - Component for displaying a list of trucks with filtering capabilities
- **[3D Configurator](./3d-configurator)** - Interactive 3D vehicle configurator component

## Component Standards

### Naming Convention
Components follow the convention of `[component-name].html` for template files and `[component-name].css` for styling.

### Structure
All components include:
- HTL template files
- CSS styles
- Sling Models for business logic
- Dialog configurations for authoring
- Edit Configuration for author experience

### Design Principles
- Responsive design across all devices
- Accessibility compliance (WCAG 2.1 AA standards)
- Performance optimization
- Consistent styling with the design system
- Reusable and configurable components

### Component Lifecycle
1. **Development** - Component created with all necessary files
2. **Testing** - Component tested for functionality, performance, and accessibility
3. **Documentation** - Component documented with usage examples
4. **Deployment** - Component deployed to AEM instance

## Getting Started

To implement any component in your project:

1. Ensure dependencies are properly configured
2. Include necessary client libraries
3. Configure component properties as needed
4. Test across all supported browsers and devices

For detailed implementation instructions, refer to the individual component documentation pages.
