## CSS Styles
The component uses the following CSS classes:

### Main Container
- `.footer-v1` - Main footer container with dark background, white text, and relative positioning
- `.footer-bg` - Background element with radial gradients and pointer events disabled
- `.footer-grid` - Grid layout container for footer sections with responsive columns

### Brand Section
- `.footer-brand` - Brand information container
- `.footer-logo` - Logo styling with large font size and heavy weight
- `.vol` - Styling for the first part of the logo with accent color
- `.trux` - Styling for the second part of the logo with white color
- `.footer-tagline` - Tagline text styling with opacity and limited width

### Link Sections
- `.footer-links` - Container for link groups
- `.footer-links h4` - Link group titles with accent color
- `.footer-links ul` - Unstyled list container
- `.footer-links li` - Individual link items with spacing
- `.footer-links a` - Link styling with hover effect and color changes

### Newsletter Section
- `.footer-newsletter` - Newsletter subscription container
- `.footer-form` - Form layout for email subscription
- `.footer-form input` - Email input field styling
- `.footer-form button` - Newsletter submit button with accent background

### Bottom Section
- `.footer-bottom` - Bottom footer area with legal information
- `.legal-links a` - Legal links styling with hover effects

### Responsive Effects
- Media queries adjust layout for different screen sizes
- Responsive grid that changes from 5 columns to 2 columns to 1 column
- Mobile-specific layout for bottom section

## Sling Model

The FooterV1Model provides the data for the footer component:

### Java Class Structure

```java
package com.voltrux.aem.core.models;

import org.apache.sling.api.SlingHttpServletRequest;
import org.apache.sling.models.annotations.Model;
import org.apache.sling.models.annotations.injectorspecific.ValueMapValue;
import org.apache.sling.models.annotations.Default;

import javax.annotation.PostConstruct;
import java.util.List;

@Model(adaptables = SlingHttpServletRequest.class, adapters = FooterV1Model.class, resourceType = "voltrux/components/content/footer/v1")
public class FooterV1Model {

    @ValueMapValue
    @Default(values = "Redefining Electric Luxury Trucks")
    private String tagline;

    private List<LinkGroup> linkGroups;

    @PostConstruct
    protected void initModel() {
        // Initialize link groups from component configuration
        loadLinkGroups();
    }

    // Getters
    public String getTagline() { return tagline; }
    public List<LinkGroup> getLinkGroups() { return linkGroups; }

    private void loadLinkGroups() {
        // Logic to load link groups from content
    }

    // Inner class for link groups
    public static class LinkGroup {
        private String title;
        private List<Link> links;

        // Getters and setters
        public String getTitle() { return title; }
        public List<Link> getLinks() { return links; }
    }

    public static class Link {
        private String text;
        private String path;

        // Getters and setters
        public String getText() { return text; }
        public String getPath() { return path; }
    }
}
```

## Previewing in AEM

To properly preview the Footer component:

### Prerequisites
- AEM instance running (6.5 or later)
- Voltrux site structure deployed

### Steps to Preview

1. **Deploy the Component**
   ```bash
   # Use your build tool to deploy to AEM
   mvn clean install -PautoInstallPackage
   ```

2. **Create or Navigate to a Page**
   - Go to your site in AEM (e.g., `http://localhost:4502/sites`)
   - Create a new page or edit an existing one

3. **Add the Component**
   - Enter edit mode by clicking the "Edit" button
   - Drag the "Footer V1" component from the component panel to your page
   - Configure using the dialog:
     - Set the "Tagline" text
     - Add link groups with titles and links

4. **Preview the Component**
   - Click "Page Information" → "View as Published" or use Preview mode
   - Verify responsive behavior by resizing the browser window

### Troubleshooting Preview Issues
- If styling is missing: Ensure client libraries are properly configured
- If links don't work: Verify that link paths are correctly configured
- If tagline doesn't appear: Check that the property is properly saved
