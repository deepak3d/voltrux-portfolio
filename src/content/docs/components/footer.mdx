### footer.html

```html
<sly data-sly-use.model="com.voltrux.aem.core.models.FooterModel"/>

<footer class="footer" data-component="footer">
    <div class="container">
        <div class="footer-grid">
            <div class="footer-brand">
                <h2>
                    <span class="vol">VOL</span><span class="trux">TRUX</span>
                </h2>
                <p class="tagline">${model.tagline @ context='html'}</p>
            </div>
            
            <div class="footer-newsletter">
                <h4>${model.newsletterTitle @ context='html'}</h4>
                <form action="${model.newsletterFormAction @ context='uri'}" method="post" class="newsletter-form">
                    <input type="email" name="email" placeholder="Your email" required />
                    <button type="submit">${model.newsletterButtonText @ context='html'}</button>
                </form>
                <small>${model.newsletterDisclaimer @ context='html'}</small>
            </div>
            
            <sly data-sly-repeat.group="${model.linkGroups}">
                <div class="footer-links">
                    <h4>${group.title @ context='html'}</h4>
                    <ul>
                        <sly data-sly-repeat.link="${group.links}">
                            <li><a href="${link.path @ context='uri'}">${link.text @ context='html'}</a></li>
                        </sly>
                    </ul>
                </div>
            </sly>
        </div>
        
        <div class="footer-bottom">
            <div class="footer-legal">
                <p>&copy; ${model.copyrightYear @ context='html'} ${model.companyName @ context='html'}. All rights reserved.</p>
            </div>
            <div class="legal-links">
                <a href="${model.privacyPolicyPath @ context='uri'}">Privacy Policy</a>
                <a href="${model.termsOfServicePath @ context='uri'}">Terms of Service</a>
            </div>
        </div>
    </div>
</footer>
```

## CSS Styles

The component uses the following CSS classes:

### Main Container
- `.footer` - Main footer container with dark background and appropriate padding
- `.container` - Centered container with max-width constraint

### Brand Section
- `.footer-brand` - Container for brand information with centered layout
- `.vol` - Styling for the "VOL" portion of the logo in accent color
- `.trux` - Styling for the "TRUX" portion of the logo in white
- `.tagline` - Tagline text styling with muted appearance

### Newsletter Section
- `.footer-newsletter` - Container for newsletter signup with centered layout
- `.newsletter-form` - Form styling with email input and submit button
- `input[type="email"]` - Email input field styling
- `button` - Submit button styling with accent background

### Link Sections
- `.footer-links` - Container for individual link groups
- `h4` - Link group title styling
- `ul` - List container for links
- `li` - Individual link item styling
- `a` - Link styling with hover effects

### Bottom Section
- `.footer-bottom` - Bottom section with legal information and links
- `.footer-legal` - Copyright text container
- `.legal-links` - Container for legal links like privacy policy and terms

### Responsive Effects
- On mobile screens, the layout changes to a single column
- Link groups stack vertically instead of horizontal arrangement
- Spacing adjusts for smaller screens
- Elements maintain proper touch targets for mobile interaction

## Previewing in AEM

To properly preview the Footer component:

### Prerequisites
- AEM instance running (6.5 or later)
- Voltrux site structure deployed
- Sample content created

### Steps to Preview

1. **Deploy the Component**
   ```bash
   # Deploy using your build tool
   mvn clean install -PautoInstallPackage
   ```

2. **Add Component to Page**
   - Navigate to your site in AEM (e.g., http://localhost:4502/sites)
   - Edit a page that supports the footer component
   - Drag the "Footer" component from the component panel to your page

3. **Configure Component**
   - Set the "Tagline" text
   - Define "Link Groups" with titles and associated links
   - Configure newsletter settings

4. **Preview the Component**
   - Use Preview mode to see how the component renders
   - Test responsive behavior by resizing the browser window
   - Verify that all links work properly

### Troubleshooting Preview Issues
- If styling is not appearing: Check that client libraries are properly configured
- If links don't work: Verify that link paths are correctly configured in the dialog
- If newsletter form doesn't submit: Check that the form action path is correct

## Sling Model

The Footer component uses the following Sling Model:

### Java Class Definition
```java
@Model(adaptables = SlingHttpServletRequest.class, adapters = FooterModel.class, resourceType = "voltrux/components/structure/footer")
public class FooterModel {
    @ValueMapValue
    private String tagline;

    @ValueMapValue
    private String newsletterTitle;

    @ValueMapValue
    private String newsletterButtonText;

    @ValueMapValue
    private String newsletterDisclaimer;

    @ValueMapValue
    private String copyrightYear;

    @ValueMapValue
    private String companyName;

    @ValueMapValue
    private String newsletterFormAction;

    @ValueMapValue
    private String privacyPolicyPath;

    @ValueMapValue
    private String termsOfServicePath;

    // List of LinkGroup objects with title and List<Link> inside each group
    private List<LinkGroup> linkGroups;

    // Getters for all properties
    public String getTagline() { return tagline; }
    // ... other getters
}

public class LinkGroup {
    private String title;
    private List<Link> links;

    // Getters
    public String getTitle() { return title; }
    public List<Link> getLinks() { return links; }
}

public class Link {
    private String text;
    private String path;

    // Getters
    public String getText() { return text; }
    public String getPath() { return path; }
}
```
