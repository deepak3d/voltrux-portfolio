## Dialog Configuration

The component includes a comprehensive dialog for content authors, allowing them to customize the grid content and truck items. Here's an interactive preview of the Touch UI dialog:

<TruckGridDialog />

The Touch UI dialog provides an intuitive interface for content authors to:

- Set the section title and subtitle
- Add multiple truck items with primary and secondary images
- Configure titles and captions for each truck
- Easily reorder items using the up/down buttons

### Desktop View
- Section title and subtitle are prominently displayed at the top
- Truck cards are displayed in a responsive grid format (typically 3 columns)
- Each truck card features dual images (front and side views) in a horizontal layout
- Truck information is displayed below the images with title and caption

### Mobile View
- Truck grid adjusts to 1 column for better readability on small screens
- Dual images stack vertically within each card
- Content remains readable and accessible

## Component Structure

The Truck Grid component consists of several key files that define its functionality and appearance:

### .content.xml

```xml
<?xml version="1.0" encoding="UTF-8"?>
<jcr:root xmlns:cq="http://www.day.com/jcr/cq/1.0" xmlns:jcr="http://www.jcp.org/jcr/1.0"
    jcr:description="Voltrux Motors Truck Grid Component"
    jcr:primaryType="cq:Component"
    jcr:title="Grid Component"
    componentGroup="Voltrux Motors - Content"/>
```

### truckgrid.html

```html
<sly data-sly-use.grid="com.voltrux.aem.core.models.TruckGridModel">
    <section class="truck-grid" data-cq-data-path="${resource.path}">
        <div class="truck-grid__container">
            <sly data-sly-test="${grid.title}">
                <h2 class="truck-grid__title">${grid.title}</h2>
            </sly>
            <sly data-sly-test="${grid.subtitle}">
                <p class="truck-grid__subtitle">${grid.subtitle}</p>
            </sly>
            <div class="truck-grid__grid">
                <sly data-sly-list.item="${grid.items}">
                    <div class="truck-grid__card">
                        <div class="truck-grid__images">
                            <sly data-sly-test.primaryPath="${item.imagePrimary}">
                                <img src="${primaryPath @ extension='jpg'}" alt="${item.title @ format='Primary Image for {0}' || 'Primary Image'}" class="truck-grid__image" />
                            </sly>
                            <sly data-sly-test="${!item.imagePrimary}">
                                <img src="/content/dam/voltrux/fallback-image.jpg" alt="Fallback Primary Image" class="truck-grid__image" />
                            </sly>
                            <sly data-sly-test.secondaryPath="${item.imageSecondary}">
                                <img src="${secondaryPath @ extension='jpg'}" alt="${item.title @ format='Secondary Image for {0}' || 'Secondary Image'}" class="truck-grid__image" />
                            </sly>
                            <sly data-sly-test="${!item.imageSecondary}">
                                <img src="/content/dam/voltrux/fallback-image.jpg" alt="Fallback Secondary Image" class="truck-grid__image" />
                            </sly>
                        </div>
                        <div class="truck-grid__content">
                            <sly data-sly-test="${item.title}">
                                <h3 class="truck-grid__card-title">${item.title}</h3>
                            </sly>
                            <sly data-sly-test="${item.caption}">
                                <p class="truck-grid__card-caption">${item.caption}</p>
                            </sly>
                        </div>
                    </div>
                </sly>
            </div>
        </div>
    </section>
</sly>
```

## CSS Styles

The component uses the following CSS classes:

### Main Container
- `.truck-grid` - Main container with consistent padding
- `.truck-grid__container` - Centered container with max-width and responsive padding
- `.truck-grid__title` - Section title with XXL font size, bold weight, and primary color
- `.truck-grid__subtitle` - Section subtitle with large font size and muted color

### Grid Layout
- `.truck-grid__grid` - Grid layout for displaying truck cards with responsive columns

### Truck Cards
- `.truck-grid__card` - Individual truck card with background, rounded corners, overflow hidden and shadow
- `.truck-grid__images` - Container for truck images with flex layout
- `.truck-grid__image` - Individual truck image with 50% width and object fit
- `.truck-grid__content` - Content area with padding and text centering

### Content Elements
- `.truck-grid__card-title` - Card title with large font size, bold weight, and primary color
- `.truck-grid__card-caption` - Card caption with medium font size and muted color

### Responsive Effects
- Uses responsive grid layout that adapts to screen size
- Dual images stack vertically on mobile
- Truck cards adjust from multiple columns on desktop to single column on mobile

## Previewing in AEM

To properly preview the Truck Grid component:

### Prerequisites
- AEM instance running (6.5 or later)
- Voltrux site structure deployed
- Sample truck inventory created

### Steps to Preview

1. **Deploy the Component**
   ```bash
   # Use your build tool to deploy to AEM
   mvn clean install -PautoInstallPackage
   ```

2. **Create or Navigate to a Page**
   - Go to your site in AEM (e.g., `http://localhost:4502/sites`)
   - Create a new page or edit an existing one

3. **Add the Component**
   - Enter edit mode by clicking the "Edit" button
   - Drag the "Truck Grid" component from the component panel to your page
   - Configure using the dialog:
     - Set the "Section Title"
     - Add "Section Subtitle"
     - Add truck items with primary and secondary images

4. **Preview the Component**
   - Click "Page Information" → "View as Published" or use Preview mode
   - Verify responsive behavior by resizing the browser window

### Troubleshooting Preview Issues
- If no trucks appear: Verify that your truck items have been added with required images
- If images don't load: Check that image paths are correct and files exist
- If styling is missing: Ensure client libraries are properly configured
